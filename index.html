What is Rownd.js?

Rownd.js is a quick and simple Front-end framework. Lightweight and easier to build with than other MVC frameworks such as Angular.js or Ember.js.
It uses Ractive.js at it's heart which makes rendering pages instant and allows two way data-binding so you can be as dynamic as possible and bring the best UX to your users.

Set Up

Setting up Rownd.js can be very simple, just download the Start up kit and you're off.

Of course you don't have to use the Start up kit. You can just put it all together yourself. Just donwload Rownd and the Rownd template compiler. Put it all together with your favorite build tool and build something cool.

Installing with npm

npm install rownd --save

Base Structure

To set up your Rownd.js application, you're going to need to create a couple files and then link them up into your index.html. You can do this with a build tool or just link all your files up, which ever you prefer. The first file you're going to need is the Routes.js file. In here you will start the Rownd instance and link up all your routes. This file should always be the first file to be linked in your index.html.

To start Rownd, you can use the Rownd.start() function. You can pass an object into this function to change the default parameters if you wish to. After that you're going to need to create your first route. To create a route you use the Rownd.createRoute() function within which you need to specifiy your path and controller for that path. 

            
  var rnd = Rownd.start({
    'debug': true,
    'hideInfo': false,
    'showVersion': true
  });

  rnd.createRoute({
    'path': '',
    'controller': 'index'
  });

The next file you're going to need to create is your controller file. In here you can create your first controller using the Rownd.createController(). When creating a controller you need to specify the name and the controller object. 

            
  rnd.createController('index', function() {

    // Specifiy the initial template
    'view': {
      'template': 'index'
    },

    // All your logic goes here
    'controller': function(routeData){
    },

    //  All the template logic functions
    'actions': {
      'concat': funtion(wordOne, wordTwo) {
        return wordOne + wordTwo;
      }
    }
  });

The last step is to create a view using handlebars. To compile to handlebars files to Ractive templates we use the Rownd-Ractive-compiler. Which can be obtained through npm.

Routes

In Rownd, routes are used to load individual pages and controllers into existence. Soon we will be adding in hierarchical routes but right now they're just plain and simple.

Creating Routes

To create a route you will need to use the Rownd.createRoute() function. To use this function you simply add it to your routes file and pass it an object with two parameters; controller and path. The controller item must be the same as the name of the controller you want to use and the path is the url you would like to use for this new page.

404 Pages

To create a 404 page you will need to utilize the Rownd.routeNotFound() function. This function will fire whenever a route cannot be found. You can run anything you want inside this function but we suggest serving up a nice 404 page.

Templates

Templates in Rownd are handlebar files that are rendered by Ractive into html. Because they are rendered by ractive they are extremely quick to render. They also allow for two way data-binding between the controller and the template.

generateTemplate()

If you want to load in a partial template or render a whole new template on the current page you can use the Rownd.generateTemplate() function. generateTemplate() only requires an object as it's parameter with the following contents.

  • template - Name of the new template
  • outlet - The location of the template. Leave out if you want to replace the whole document with the new template
  • data - If you want to add any intial data when loading the template
  • append - Append new template to the selected outlet or overwrites the outlet specified

Controllers

Controllers are where most of the logic for your application will be based. Some small logic may be in the template actions or in your helper library. Creating a controller is simple, just use the Rownd.createController() function and match up the name of the controller with the name used in the Rownd.createRoute() function. Below is an example controller: 

            
  rwnd.createController('index', function() {

    // Specifiy the initial template
    'view': {
      'template': 'index'
    },

    // All your logic goes here
    'controller': function(routeData){
    },

    //  All the template logic functions
    'actions': {
      'concat': funtion(wordOne, wordTwo) {
        return wordOne + wordTwo;
      }
    }
  });

Route Data

To get the current route data add the parameter routeData to the controller function and then you can use that variable to get the current path and the name of the controller you're currently using.

View Object

The view object is what Rownd uses to load in a template when a new route is hit and the controller is fired. This view object is ran through the generateTemplate function which means you can add data to the template by adding an object called data inside this view object.

Actions Object

The actions object is used to give functionality to the template. Inside here you can write functions that are accessable from the template.

Using Data-Binding

To use data binding in the controller you can use the Ractive get and set functions. Because the view is directly rendered with Ractive we can use all of Ractive's instance methods. Which you can find all the documentation here.

Helpers

Instead of using directives and injecting them like Angular.js, in Rownd you have the Helpers object. The Helpers object contains a range of functions created by simply adding an item to the Helpers object. Helpers can be used in any controller as many times as you want. Below is an example of creating a Helper using the Rownd instnace we set up at the top. 

            
  rnd.Helpers.stringValidator = function(stringOne, stringTwo){
    return stringOne === stringTwo;
  };
