What is Rownd.js?

Rownd.js is a quick and simple Front-end framework. Lightweight and easier to build in than other MVC frameworks such as Angular.js or Ember.js.
It uses Ractive.js at it's heart which makes rendering pages instant and allows two way data-binding so you can be as dynamic as possible with your interfaces.

Set Up

Setting up Rownd can be done either with the Start Up Kit or by getting it directly from npm. Using the Start Up Kit sets everything up for you. If you're setting this up without the Start Up kit then follow the guide below.

If you haven't used Rownd.js before we highly advise you use the start up kit to get started. The start up kit is just the bare bones you need to get started with a coulpe example code so you can see where everything goes.

Install

To install Rownd.js simply install it via npm using the command npm install rownd --save

Base Structure

To get Rownd.js working you're going to need to set up an index route, controller and view. First start by creating a Routes.js and create a new isntance of Rownd with your own config using the start function - var rwnd = Rownd.start({debug: false});. This is how you change your config options, just give a config a value of your choosing and add it to the object in the start function.

Now that you have the start function going we need to create a route. To create a route type out rwnd.createRoute({'controller' : 'index','path' : ''});. When creating a route you will need to specify the path of the route (in this case empty string denotes root). You also need to specify the controller that will be used for that route, here we've used index as the name for the index controller.

Your route is set up but now you need a controller, to do that you'll need to use the createController function and pass it a name and an object which contains a view object and a controller object. E.g. rwnd.createController('index', {'view': {'template': 'index'},'controller': function(){}});. This will create an index controller for you. Inside the view object you specifiy a template which we've called index. This needs to match up to a handlebars file with the same name. Then we have a function called controller which will contain all of the logic for that route.

The final step is to make the handlebars file called index.html so that it matches the name in the view.

Routes

In Rownd, routes are used to load individual pages and controllers into existence. Soon we will be adding in hierarchical routes but right now they're just plain and simple.

Creating Routes

To create a route add you need to use the rwnd.createRoute() function. To use this function you simply add it to your routes file and pass it an object with two parameters; controller and path. The controller item must be the same as the name of the controller you want to use and the path is the url you would like to use for this new page.

404 Pages

To create a 404 page you will need to utilize the rwnd.routeNotFound() function. This function will fire whenever a route cannot be found. You can run anything you want inside this function but we suggest serving up a nice 404 page.

Templates

Templates in Rownd are handlebar files that are rendered by Ractive into html. Because they are rendered by ractive they are extremely quick to render. They also allow for two way data-binding between the controller and the template.

generateTemplate()

If you want to load in a partial template or render a whole new template on the current page you can use the rwnd.generateTemplate() function. generateTemplate() requires one parameter which is an object which will hold all of the parameters needed.

  • template - Name of the new template
  • outlet - The location of the template. Leave blank if you want to replace the current template with the new one
  • data - Any data to bind to the template. Must be valid JavaScript
  • append - Uses the append value in Ractive templates

Controllers

Controllers are where most of the logic for your application will be based. Some small logic may be in the template or in the use of helpers. To create a controller for use with a route there is the rwnd.createController() function. This function takes two parameters, the first being a String containing the name of the controller to reference it in the rwnd.createRoute() function and then an Object which will contain two items required and one optional item.

The first required item inside the object is a view Object which is essentially uses the rwnd.generateTemplate() function to load the first view for your controller. The second item is a function with the key controller. This function has one parameter which is the route data of the current route. The last item is the optional actions object which can contain a range of functions which can be added to the template set in the view object.

Route Data

To get the current route data add the parameter routeData to the controller function and then you can use that variable to get the current path and the name of the controller you're currently using.

View Object

The view object is what Rownd uses to load in a template when a new route is hit and the controller is fired. This view object is ran through the generateTemplate function which means you can add data to the template by adding an object called data inside this view object.

Actions Object

The actions object is used to give functionality to the template. Inside here you can write functions that are accessable from the template.

Using Data-Binding

To use data binding in the controller you can use the Ractive get and set functions. Because the view is directly rendered with Ractive we can use all of Ractive's instance methods. Which you can find all the documentation here.
