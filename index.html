Rownd.js A super lightweight Front-End Framework

What is Rownd.js?

Rownd.js is a quick and simple Front-end framework. Lightweight and easier to build in than other MVC frameworks such as Angular.js or Ember.js.
It uses Ractive.js at it's heart which makes rendering pages instant.

Set Up

Setting up Rownd can be done either with the Start Up Kit or by getting it directly from npm. Using the Start Up Kit sets everything up for you. If you're setting this up without the Start Up kit then follow the guide below.

Install

To install Rownd.js simply install it via npm using the command npm install rownd --save

Composing

To configure Rownd.js you're going to need to set up an index route, controller and view. First start by creating a Routes.js and create a new isntance of Rownd with your own config using the start function - var App = Rownd.start({debug: false});. This is how you change your config options, just give a config a value of your choosing and add it to the object in the start function.

Now that you have the start function going we need to create a route. To create a route type out App.createRoute({'controller' : 'index','path' : ''});. When creating a route you will need to specify the path of the route (in this case empty string denotes root). You also need to specify the controller that will be used for that route, here we've used index as the name for the index controller.

Your route is set up but now you need a controller, to do that you'll need to use the createController function and pass it a name and an object which contains a view object and a controller object. E.g. App.createController('index', {'view': {'template': 'index'},'controller': function(){}});. This will create an index controller for you. Inside the view object you specifiy a template which we've called index. This needs to match up to a handlebars file with the same name. Then we have a function called controller which will contain all of the logic for that route.

The final step is to make the handlebars file called index.html so that it matches the name in the view.
